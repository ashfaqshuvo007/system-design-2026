# 🛠 Curated Topics for Low‑Level Design (LLD)

LLD interviews assess your ability to write clean, extensible code using OOP and design patterns.

## 1. SOLID Principles

understand each principle and be able to refactor a simple class diagram.

## 2. Common Design Patterns

- **Creational**: Factory, Singleton, Builder

- **Structural**: Adapter, Decorator, Facade

- **Behavioral**: Observer, Strategy, State

## 3. Object‑Oriented Design Problems

- Parking lot, elevator system, vending machine, ATM

- Cache (LRU), rate limiter, task scheduler

- Movie ticket booking, restaurant management

## 🧠 How to Tackle “LLD / OOD” in Mid‑Size Company Interviews

Many mid‑size companies also ask one low‑level design round. During weeks 5‑7, pick one object‑oriented design problem per week (e.g., parking lot, elevator, ATM, LRU cache). Spend one 2‑hour session designing it with class diagrams, then code it test‑driven with SOLID principles. Containerise it and add a CI test suite. Use Alex Xu’s latest “Object‑Oriented Design Interview” book for model solutions.

**Example schedule**:

- **Week 5**: Parking Lot (creational + strategy patterns)

- **Week 6**: Elevator System (state pattern)

- **Week 7**: LRU Cache (design a generic cache using hash map + doubly linked list)
