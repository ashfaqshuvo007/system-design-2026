# 📚 Curated Topics for High‑Level Design (HLD)

Master these concepts in order; each is a building block for the next.

## Scalability Basics

- Horizontal vs. vertical scaling

- Load balancing (round‑robin, least‑connections, consistent hashing)

- Caching strategies (read‑through, write‑through, write‑behind)

- Content Delivery Networks (CDNs)

## Database Design & Trade‑offs

- SQL vs. NoSQL (when to use which)

- Sharding & partitioning strategies

- Replication (master‑slave, master‑master)

- Indexing best practices

## Communication & APIs

- REST, gRPC, GraphQL – when and why

- Message queues (Kafka, RabbitMQ) vs. pub/sub

- Event‑driven architecture (CQRS, Event Sourcing)

## Distributed Systems Core

- CAP theorem (and its real‑world implications)

- Consistency models (strong, eventual, causal)

- Consensus algorithms (Raft, Paxos) – high‑level understanding

- Consistent hashing, quorum reads/writes

## Reliability & Fault Tolerance

- Circuit breaker, retry with backoff, bulkhead

- Idempotency & exactly‑once semantics

- Disaster recovery (backup, replication, multi‑region failover)

## Microservices Patterns

- Service discovery, API gateway

- Saga pattern for distributed transactions

- Sidecar pattern, Strangler Fig for migration

## Security & Observability

- OAuth2, JWT, rate limiting

- Logging, metrics, tracing (e.g., ELK, Prometheus, Jaeger)
